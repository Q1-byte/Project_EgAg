## 핵심 용어 정의

- **1획(stroke)**: 사용자가 마우스/터치를 누르고(pointerdown) 떼는(pointerup) 한 번의 연속 드로잉
- **1턴**: 사용자 1획 + AI 1획으로 구성
- **세션**: 캔버스 진입부터 '완성' 또는 '저장'까지의 전체 드로잉 과정
- **토큰**: 그림 1회 시작에 필요한 재화. 신규 가입 시 3개 무료 지급, 이후 결제로 충전

## 캔버스 핵심 로직

### 턴 흐름
1. 사용자 pointerdown → 새 stroke 객체 생성
2. pointermove → points 배열에 좌표 추가
3. pointerup → 1획 완료 → strokes 배열에 저장
4. isAiTurn = true → `POST /api/canvas/{id}/stroke` 호출
5. 서버: canvasImage(base64) → GPT-4o Vision → AI 획 좌표 반환
6. 프론트: AI 획을 800ms 애니메이션으로 점진적 렌더링
7. isAiTurn = false → 사용자 차례

### 턴 제한
- **소프트 리밋 (15턴)**: AI 코멘트 "거의 완성이에요!" + 완성 버튼 pulse 애니메이션
- **하드 리밋 (20턴)**: 자동으로 색칠 단계 전환, 추가 획 입력 불가

### 색상 구분
- 사용자 획: 검정 (#000000)
- AI 획: 파랑 (#3B82F6)

### Konva.js 핵심
```tsx
<Stage ref={stageRef} width={800} height={600}
  onPointerDown={handlePointerDown}
  onPointerMove={handlePointerMove}
  onPointerUp={handlePointerUp}
  style={{ touchAction: "none" }}
>
  <Layer>
    {strokes.map(stroke => (
      <Line key={stroke.id} points={stroke.points}
        stroke={stroke.color} strokeWidth={stroke.strokeWidth}
        lineCap="round" lineJoin="round" tension={0.5} />
    ))}
  </Layer>
</Stage>
```

### 이미지 추출 (AI 전송 / 저장)
```ts
const dataURL = stageRef.current.toDataURL({ pixelRatio: 1 }); // base64 PNG
```

## AI Agent 구현 (LangChain4j)

### 공통 구조
3개 Agent 모두 LangChain4j의 `ChatLanguageModel`을 통해 GPT-4o Vision API를 호출하고, 프롬프트만 다름.
Spring Boot Starter 자동 설정으로 `ChatLanguageModel` 빈이 주입됨.

```java
// OpenAiClient.java — 공통 호출 클래스
@Component
@RequiredArgsConstructor
public class OpenAiClient {
    private final ChatLanguageModel chatLanguageModel;

    public String chat(List<ChatMessage> messages) {
        return chatLanguageModel.chat(messages).aiMessage().text();
    }
}
```

### Stroke Agent
- 호출 시점: 매 턴 (사용자 1획 완료 후)
- 입력: 캔버스 이미지 (base64) + 이전 대화 기록
- 출력: `{ points: [x1,y1,x2,y2,...], comment: "잘 그리고 있어요! 🎨", done: false }`
- 대화 기록: artworks.ai_context에 JSON으로 저장, 10턴 윈도우 (오래된 것 잘라냄)

### Theme Agent
- 호출 시점: 첫 획 완료 직후 (1회)
- 입력: 캔버스 이미지
- 출력: `{ suggestions: ["고양이", "해바라기", "로켓"] }`

### Color Agent
- 호출 시점: '완성' 버튼 클릭 후 (1회)
- 입력: 완성된 선화 이미지
- 출력: `{ colorRegions: [{ area: "배경", color: "#87CEEB" }, ...] }`

## 인증 흐름

### JWT 구조
- Access Token: 15분 만료, Authorization 헤더에 Bearer로 전송
- Refresh Token: 7일 만료, DB(refresh_tokens)에 해시 저장
- 토큰 갱신: access 만료 시 refresh로 자동 갱신 (Axios 인터셉터)

### 회원가입
- 이메일 + 비밀번호 (bcrypt) + 닉네임 (2~12자)
- 이메일 인증 없음 — 가입 즉시 사용 가능
- Google OAuth 원클릭 가입
- 이용약관 + 개인정보처리방침 동의 필수

## API 공통 규칙

- Base URL: `/api`
- 인증: `Authorization: Bearer <accessToken>` (🔒 표시된 엔드포인트)
- Content-Type: application/json (파일 업로드 시 multipart/form-data)
- 페이지네이션: 커서 기반 (?cursor=&limit=20)
- 에러 응답: `{ error: { code: "TOKEN_INSUFFICIENT", message: "토큰이 부족합니다" } }`
- Rate Limit: 일반 100 req/min, AI 턴당 1초, 인증 10 req/min per IP

### HTTP 상태 코드
- 200 성공 / 201 생성 / 400 유효성 실패 / 401 인증 실패 / 402 토큰 부족 / 403 권한 없음 / 404 없음 / 409 중복 / 429 Rate Limit / 500 서버 오류

## 코딩 컨벤션

### 백엔드 (Java/Spring Boot)
- 패키지: `com.egag.*`
- Entity: 단수형 (User, Artwork, Like)
- Repository: `~Repository` (UserRepository)
- Service: `~Service` (AuthService)
- Controller: `~Controller` (AuthController)
- DTO: `~Request` / `~Response` (LoginRequest, AuthResponse)
- 예외: `~Exception` (TokenInsufficientException)
- REST 경로: 복수형 소문자 (/api/artworks, /api/users)

### 프론트엔드 (TypeScript/React)
- 컴포넌트: PascalCase (DrawingCanvas.tsx)
- 훅: camelCase use~ (useCanvas.ts)
- API 함수: camelCase (getArtwork, createStroke)
- 상태 스토어: use~Store (useAuthStore)
- 타입: PascalCase (ArtworkResponse, StrokeData)
- 페이지: PascalCase (Canvas.tsx, Explore.tsx)
