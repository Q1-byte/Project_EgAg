# EgAg — 개발환경 셋업 가이드

> 프로젝트를 처음 받았을 때 이 문서를 따라하면 됩니다.

---

## 1. 사전 설치 (필수)

아래 프로그램이 없으면 먼저 설치해주세요.

| 프로그램 | 버전 | 설치 링크 |
|---------|------|----------|
| **Java (JDK)** | 25 | https://jdk.java.net/25/ |
| **Node.js** | 20 이상 | https://nodejs.org/ |
| **MySQL** | 8.x | https://dev.mysql.com/downloads/installer/ |
| **MySQL Workbench** | 최신 | https://dev.mysql.com/downloads/workbench/ |
| **IntelliJ IDEA** | 권장 | https://www.jetbrains.com/idea/ |

> Gradle은 프로젝트에 포함된 `gradlew`가 자동으로 다운받으므로 별도 설치 불필요.

---

## 2. 프로젝트 클론

```bash
git clone <레포지토리 URL>
cd egag
```

---

## 3. MySQL 데이터베이스 생성

### MySQL Workbench 사용
1. Workbench 열기
2. `+` 버튼으로 새 Connection 생성

| 항목 | 값 |
|------|-----|
| Connection Name | egag |
| Hostname | 127.0.0.1 |
| Port | 3306 |
| Username | root |
| Password | 1234 |

3. 연결 후 아래 SQL 실행:

```sql
CREATE DATABASE egag;
```

> 테이블은 서버 실행 시 JPA가 자동 생성합니다. 직접 만들 필요 없음!

---

## 4. 백엔드 실행

프로젝트 루트(egag/)에서:

```bash
# 빌드 확인
./gradlew build

# 서버 실행
./gradlew bootRun
```

- 실행되면 `http://localhost:8080` 에서 백엔드 동작
- 최초 실행 시 MySQL에 11개 테이블 자동 생성됨
- Workbench에서 `egag` 스키마 새로고침하면 테이블 확인 가능

### Windows에서 실행 시
```bash
gradlew.bat build
gradlew.bat bootRun
```

---

## 5. 프론트엔드 실행

```bash
# frontend 폴더로 이동
cd frontend

# 의존성 설치 (최초 1회)
npm install

# 개발 서버 실행
npm run dev
```

- 실행되면 `http://localhost:5173` 에서 프론트 동작
- API 호출은 자동으로 `localhost:8080`으로 프록시됨

---

## 6. 실행 확인 체크리스트

- [ ] MySQL 서버 실행 중
- [ ] `./gradlew bootRun` → 에러 없이 서버 시작됨
- [ ] `cd frontend && npm run dev` → 브라우저에서 페이지 열림
- [ ] Workbench에서 `egag` DB에 테이블 11개 생성 확인

## 7. 자주 쓰는 명령어

| 목적 | 명령어 | 위치 |
|------|--------|------|
| 백엔드 빌드 | `./gradlew build` | 루트 |
| 백엔드 실행 | `./gradlew bootRun` | 루트 |
| 프론트 의존성 설치 | `npm install` | frontend/ |
| 프론트 개발 서버 | `npm run dev` | frontend/ |
| 프론트 빌드 | `npm run build` | frontend/ |

---

## 8. 주의사항

- **application.yml**에 DB 비밀번호(`1234`)가 하드코딩되어 있음 → 본인 MySQL 비밀번호가 다르면 `src/main/resources/application.yml` 수정
- **백엔드 실행 전** MySQL 서버가 먼저 켜져 있어야 함
- **프론트 실행 전** `npm install` 한 번은 꼭 해야 함 (`node_modules`는 git에 안 올라감)
- IntelliJ에서 열 때 Lombok 플러그인 설치 + Annotation Processing 활성화 필요

## 패키지 구조 (백엔드 — 기능별)

```
src/main/java/com/egag/
├── auth/           ← 인증 (JWT, 로그인, 회원가입)
├── canvas/         ← 캔버스 + AI Agent
├── artwork/        ← 작품 + 좋아요
├── user/           ← 유저 + 갤러리 + 팔로우
├── search/         ← 검색
├── notification/   ← 알림
├── payment/        ← 결제 + 토큰
├── inquiry/        ← 문의
├── admin/          ← 관리자 + 신고
└── common/         ← 공유 (Config, Entity, Exception)
```

> 본인 담당 폴더에서 작업하면 됩니다. 다른 폴더 건드릴 일 거의 없음!
